# BIG-O 표기법

> 알고리즘분석시 필요한 기본적인 도구

---

### 두 알고리즘 비교시

1. A가 B보다 조금, 많이 빨라요 → 애매모호함
2. 프로그램을 짜서 실행속도를 비교 → 환경의존적
   1. 환경이란 컴퓨터 성능, os환경등 다양한 요소 → 애매모호 + 무의미
3. 구간별 차이 존재 → 입력이 적은구간과 많은 구간에서 성능 차이가 존재 할 수 밖에 없다.

**위 문제를 위한 대안법으로 Big-O표기법 사용**

---

### Big-O표기법 1단계: 대략적인 계산

- 수행되는 연산(산술, 비교, 대입 등)의 개수를 ‘대략적으로’ 판단

### Big-O표기법 2단계: 대장만 남긴다

- 영향력이 가장 큰 대표 항목만 남기고 삭제
- 상수는 무시한다
  <aside>

      O(2N) → O(N)
      O(N + 3) → O(N)

      생략, 상수무시의 이유는?
      의미상 빅오는 함수가 몇개의 연산을 하는지가 중요한게아니라 데이터가 늘어남에 따라서 어떤식으로 연산량이 증가하는지 알기위해 쓴다. N의 연산량이 다른 상수에 비하면 상당히 빠른속도라 상수는 영향이 미미하므로 무시가능

</aside>
